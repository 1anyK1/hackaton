# Отчет по заданию 5: реверс-инжиниринг радиоуправляемой машинки

## Цель

Необходимо перехватить управление радиоуправляемой машинкой с помощью SDR HackRF, определить протокол
оригинального пульта, найти синхросигнал корреляционным приемом, записать управляющие сигналы и реализовать
управление с клавиатуры ноутбука на C++.

На текущем этапе машинки и оригинального пульта нет, поэтому невозможно достоверно измерить несущую частоту,
модуляцию, длительности импульсов и реальные командные слова. Выполнена подготовительная часть, которую можно
проверять без объекта атаки.

## Реализованные файлы

- `CMakeLists.txt` - сборка C++17-проекта через CMake;
- `Makefile` - простой вариант сборки через `g++`;
- `src/main.cpp` - CLI-утилита с режимами `generate`, `analyze`, `tx`, `keyboard`;
- `src/protocol.*` - модель OOK/ASK-протокола, генерация кадров и декодирование PWM-битов;
- `src/correlation.*` - нормированная корреляция для поиска синхросигнала;
- `src/iq_io.*` - чтение и запись HackRF `int8` interleaved IQ;
- `README.md` - инструкция сборки, записи и запуска.

## Архитектура решения

Система разделена на четыре логических блока.

1. Генератор протокола формирует 24-битный кадр:

   `20 бит адреса + 4 бита команды`

   Такая структура выбрана как временная рабочая гипотеза для простых OOK/ASK-пультов. Реальные значения
   будут заменены после записи оригинального пульта.

2. Формирователь сигнала преобразует биты в амплитудную огибающую:

   - синхроимпульс: короткий высокий уровень и длинная пауза;
   - `0`: короткий высокий уровень и длинный низкий;
   - `1`: длинный высокий уровень и короткий низкий.

   Затем огибающая записывается как комплексный IQ-сигнал для HackRF: `I = envelope`, `Q = 0`.

3. Корреляционный приемник строит амплитудную огибающую из IQ:

   `env[n] = sqrt(I[n]^2 + Q[n]^2)`

   Далее выполняется нормированная корреляция с шаблоном синхроимпульса. Пики выше порога считаются началами
   кадров. После каждого найденного синхросигнала декодируются 24 PWM-бита.

4. CLI-слой предоставляет режим клавиатурного управления. Каждая клавиша генерирует нужную IQ-посылку и
   отправляет ее через `hackrf_transfer`.

## Команды запуска

Сборка:

```bash
cd /home/anyk/Hakaton/task5
make
```

Генерация тестовой команды:

```bash
./build/rc_car_tool generate forward generated/forward.iq
```

Анализ записи пульта:

```bash
./build/rc_car_tool analyze captures/remote_27145k_s8.iq 0.78
```

Передача команды:

```bash
./build/rc_car_tool tx forward 27145000 2000000
```

Клавиатурное управление:

```bash
./build/rc_car_tool keyboard 27145000 2000000
```

## Что уже можно проверить без машинки

- Корректность структуры проекта и сборки C++.
- Генерацию IQ-файлов команд.
- Работу корреляционного поиска на синтетических IQ-файлах, сгенерированных самой программой.
- Наличие HackRF через `hackrf_info`.
- Возможность записывать IQ-дампы через `hackrf_transfer`.

## Что нужно сделать после появления машинки

1. Найти несущую частоту пульта по маркировке, спектру или `hackrf_sweep`.
2. Записать IQ-дампы для каждой кнопки: вперед, назад, влево, вправо, стоп.
3. По огибающей измерить длительность базового импульса, синхропаузу, количество повторов и способ кодирования
   `0`/`1`.
4. Настроить поля `ProtocolConfig` в `src/protocol.h`.
5. Сравнить декодированные кадры, выделить постоянный адрес и командные биты.
6. Обновить `command_code()` и `address`.
7. Проверить передачу на минимальном усилении HackRF в лабораторных условиях.

## Ограничения текущего этапа

Реальный протокол пока не определен, потому что нет физического источника сигнала. Поэтому текущие командные
кадры являются не готовым управлением конкретной машинкой, а подготовленным инструментом для быстрого анализа и
последующей подстановки измеренных параметров.
