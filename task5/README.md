# Task 5: RC car reverse engineering with HackRF

Подготовлена C++-заготовка для задания по перехвату управления радиоуправляемой машинкой.
Без самой машинки и оригинального пульта протокол нельзя определить достоверно, поэтому текущая версия
решает ту часть, которую можно сделать заранее:

- генерирует тестовые OOK/ASK IQ-посылки для команд `forward`, `backward`, `left`, `right`, `stop`;
- ищет синхроимпульс в IQ-записи нормированной корреляцией;
- пробует декодировать 24-битный широтно-импульсный кадр после найденного синхросигнала;
- запускает передачу через HackRF CLI `hackrf_transfer`;
- содержит режим управления с клавиатуры ноутбука.

Текущая модель протокола является рабочей гипотезой, похожей на простые OOK/PT2262/EV1527-посылки:
короткий/длинный высокий импульс кодирует бит, длинная пауза используется как синхросигнал. После появления
пульта нужно заменить частоту, длительности и командные слова на измеренные.

## Сборка

Вариант с Makefile:

```bash
cd /home/anyk/Hakaton/task5
make
```

Вариант с CMake:

```bash
cd /home/anyk/Hakaton/task5
cmake -S . -B build
cmake --build build
```

В текущем окружении компилятор C++ и `cmake` могут отсутствовать; в этом случае нужно установить `g++`/`cmake`
или собрать на машине, где уже есть toolchain.

## Проверка HackRF

```bash
cd /home/anyk/Hakaton
task5/linux/2024.02.01/x86_64/bin/hackrf_info
```

Запись сигнала пульта, пример для 27.145 MHz:

```bash
mkdir -p task5/captures
task5/linux/2024.02.01/x86_64/bin/hackrf_transfer \
  -r task5/captures/remote_27145k_s8.iq \
  -f 27145000 \
  -s 2000000 \
  -a 1 \
  -l 32 \
  -g 20
```

Для реальной машинки частоту нужно найти по маркировке пульта/кварца или sweep-анализом. Часто встречаются
27 MHz, 40 MHz, 49 MHz, 315 MHz, 433.92 MHz и 2.4 GHz, но угадывать нельзя: нужна запись сигнала пульта.

## Использование

Сгенерировать IQ-файл команды без передачи:

```bash
./build/rc_car_tool generate forward generated/forward.iq
```

Найти синхросигналы и вывести декодированные биты:

```bash
./build/rc_car_tool analyze captures/remote_27145k_s8.iq 0.78
```

Передать одну команду через HackRF:

```bash
./build/rc_car_tool tx forward 27145000 2000000
```

Управление клавиатурой:

```bash
./build/rc_car_tool keyboard 27145000 2000000
```

Клавиши:

- `w` - вперед;
- `s` - назад;
- `a` - влево;
- `d` - вправо;
- `space` - стоп;
- `q` - выход.

Передача должна выполняться только в разрешенной лабораторной среде: минимальная мощность, экранирование,
аттенюаторы или согласованная тестовая схема.

## Что изменить после записи реального пульта

1. Определить несущую частоту и ширину канала.
2. Записать несколько нажатий каждой кнопки в `captures/`.
3. Подобрать `sample_rate`, `pulse_us`, длительность синхроимпульса и правила кодирования битов в
   `src/protocol.h`.
4. Сравнить декодированные биты для разных кнопок и выделить постоянную часть адреса и изменяемую часть команды.
5. Заменить `address` и `command_code()` в `src/protocol.cpp`.
6. Сгенерировать файлы команд и проверить прием на малой мощности.
